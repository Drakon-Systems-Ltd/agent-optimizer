# v0.13.0 "Agent Loop" Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Let an LLM agent (OpenClaw claw agent, Claude Code) safely drive Agent Optimizer: read findings â†’ select optimizations from a persisted plan â†’ apply transactionally â†’ verify â†’ auto-rollback on failure.

**Architecture:** The LLM stays *outside* the tool (no API keys, no data egress â€” the product's trust story is untouched). Agent Optimizer grows three things: (1) a **plan/apply contract** â€” `optimize --plan` emits a persisted, content-hashed proposal list; `optimize --apply-plan <id> --only â€¦` applies a chosen subset and refuses if the config changed since planning; (2) a **transactional apply engine** â€” multi-generation backups, post-apply validation (JSON5 reparse + auditor re-run), auto-rollback on failure, lockfile against concurrent agents; (3) an **agent surface** â€” injection-sanitized output, stable finding IDs, a SKILL.md workflow, and an OpenClaw plugin exposing the verbs as tools (apply gated behind approval).

**Tech Stack:** TypeScript (ESM, Node â‰Ą20), commander, vitest, json5. No new runtime deps.

**Non-goals:** No LLM/API calls inside agent-optimizer. No autonomous apply without a plan ID. No telemetry. No YAML config support.

**Ordering note:** Phase B (transactional engine) lands *before* the `--apply-plan` wiring so the new write path never exists without rollback.

```ts
import { createHash } from "crypto";
import { readFileSync, writeFileSync, existsSync, mkdirSync } from "fs";
import { join, resolve } from "path";
import { homedir } from "os";
import { loadConfig, expandPath } from "../utils/config.js";
import { getOptimizations } from "./openclaw/index.js";

export interface PlanProposal {
  id: string;               // "p<N>-<tag>" â€” stable within a plan
  tag: string;
  path: string;             // config path the mutation touches
  current: unknown;
  recommended: unknown;
  reason: string;
  risk: "low" | "medium" | "high";
  requiresRestart: boolean;
  info?: boolean;           // info-only proposals are never applied
}

export interface OptimizePlan {
  schemaVersion: 1;
  planId: string;           // 12-hex content hash of configHash + proposals
  createdAt: string;        // ISO
  configPath: string;
  configHash: string;       // sha256 of raw config bytes
  profile: string;
  proposals: PlanProposal[];
}

export function configHashOf(configPath: string): string {
  return createHash("sha256").update(readFileSync(expandPath(configPath))).digest("hex");
}

export function defaultPlansDir(): string {
  return join(homedir(), ".agent-optimizer", "plans");
}

export function buildPlan(configPath: string, profile: string): OptimizePlan {
  const config = loadConfig(configPath);
  if (!config) throw new Error(`Config not found or unparseable: ${configPath}`);
  const configHash = configHashOf(configPath);
  const proposals: PlanProposal[] = getOptimizations(config, profile).map((o, i) => ({
    id: `p${i + 1}-${o.tag}`,
    tag: o.tag, path: o.path, current: o.current, recommended: o.recommended,
    reason: o.reason, risk: o.risk, requiresRestart: o.requiresRestart,
    ...(o.info ? { info: true } : {}),
  }));
  const planId = createHash("sha256")
    .update(configHash).update(profile).update(JSON.stringify(proposals))
    .digest("hex").slice(0, 12);
  return {
    schemaVersion: 1, planId, createdAt: new Date().toISOString(),
    configPath, configHash, profile, proposals,
  };
}

export function savePlan(plan: OptimizePlan, plansDir = defaultPlansDir()): string {
  mkdirSync(plansDir, { recursive: true });
  const file = join(plansDir, `${plan.planId}.json`);
  writeFileSync(file, JSON.stringify(plan, null, 2));
  return file;
}

export function loadPlan(planId: string, plansDir = defaultPlansDir()): OptimizePlan | null {
  if (!/^[a-f0-9]{12}$/.test(planId)) return null; // also blocks traversal
  const file = resolve(plansDir, `${planId}.json`);
  if (!existsSync(file)) return null;
  return JSON.parse(readFileSync(file, "utf-8")) as OptimizePlan;
}
```

Note: `createdAt` is cosmetic; determinism tests compare `planId` only.

## Risks & mitigations

| Risk | Mitigation |
|---|---|
| Agent applies against changed config | `configHash` staleness guard (exit 3, re-plan) |
| Apply breaks the gateway | verify-vs-baseline + auto-rollback; backups keep 10 generations |
| Two agents apply at once | lockfile with stale-reclaim |
| Scan findings carry prompt injection into an exec-capable agent | `sanitizeUntrusted` on every scanned-content string + `untrusted: true` flags + SKILL.md hard rule |
| JSON5 comments destroyed by apply | explicit warning + original preserved in backup |
| Plugin SDK drift (the v0.11 lesson) | Task 12 Step 0: verify-first against a fresh tag worktree, no from-memory SDK calls |
